# 解構：Bus Traffic Fever!
## 核心迴圈（一圈幾秒？操作→回饋→獎勵怎麼轉？）

一圈約 20–90 秒，難度決定長度。

操作語言極度精簡：**拖動（只能單一方向）→ 空格打開 → 通道清除 → 目標車衝出口**。每一次成功拖動都有即時視覺確認（車滑進新格子、喀一聲歸位），大腦立刻感知「空間增加了」。最後一推把目標車衝出出口，音效＋動畫＋過關閃光三感合一，強烈的結束感把整段努力封包成一個記憶單元。然後「下一關」按鈕已經亮在那——迴圈立刻重啟。

## 上癮機制（用心理學拆：near-miss？歸因於己？指數獎勵？損失趨避？收集慾？）

1. **歸因於己（Skill Attribution）**：謎題讓玩家覺得「是我想通的」，不是運氣。想通瞬間 dopamine 爆發，和賭機器完全不同——你確信自己聰明，所以下一關更想證明自己。
2. **Near-miss 設計**：目標車看起來只差一格，但那一格被三台車連鎖封住。大腦被欺騙說「快了快了」，拒絕離開。最殘忍的謎題設計是讓玩家先把一個障礙推錯方向、反而堵死自己——那一刻恨得牙癢，但絕對不肯停。
3. **空間整理本能**：人類天生討厭雜亂、喜歡把東西塞進正確位置（參考 Tetris 效應）。把凌亂的格子滑整齊，觸發和「收納完成」同一塊大腦區域的滿足感。
4. **沒有失敗態**：不會「死」、不扣命、不計時（基本模式）。壓力源自「卡住的挫折感」而非「懲罰的損失感」，讓玩家更願意反覆嘗試而不是放棄。
5. **明確終點，無限延伸**：每關有清晰勝利動畫（完成！），但下一關馬上跳出。玩家沒有「我玩完了」的節點，只有「再一關」。

## 難度與節奏（怎麼讓新手活過前 15 秒、又讓老手 2 分鐘後不無聊？）

**新手生存前 15 秒**：第一關只有 2–3 台車、出口方向明顯、一次拖動即可通關。玩家在 10 秒內體驗一次完整的「想通→過關」迴圈，直覺告訴他「我學會了」，零學習曲線焦慮。目標車用高對比色（通常紅色），不需要說明書。

**讓老手 2 分鐘後不無聊**：車輛數量從 3 增加到 12–15 台；出現「必須先往反方向移才能解開」的反直覺步驟；加入 L 型長車（佔 3 格）製造更複雜的連鎖堵塞；選配時間限制模式、最少步數挑戰等第二層目標。謎題難度是三個旋鈕獨立調節：車的數量、鏈式長度、格子大小，不需要發明新機制。

## 可偷的設計（3–5 條，我們的單檔小遊戲做得到的）

1. **單軸拖曳操作**：每個物件只能橫移或縱移，觸控和滑鼠都天然流暢，不需要複雜控制設計
2. **格子座標碰撞系統**：碰撞偵測用格子 index 比對即可，完全不需要物理引擎，HTML5 Canvas 或純 CSS grid 都能做
3. **拖動時跟手、放手時 snap 歸位**：給操作「手感」，讓玩家感覺自己在真實移動東西而不是點按鈕
4. **Near-miss 謎題公式**：出口前刻意放一個看起來只差一步的障礙，讓玩家自己「發現」後面還有三步要走——這個設計讓同樣難度感覺更聰明
5. **五關精心設計勝過五十關隨機生成**：玩家不是來刷關的，是來體驗「想通瞬間」的，五個難度各異的關卡足以傳達這個快感

## 不可行的部分（原作有但我們該捨棄的，為什麼）

- **無限關卡庫**：原作幾百關甚至 AI 生成，我們做 8–10 關精品就夠，目標是體驗不是留存
- **最少步數排行 / 社交對比**：需要後端帳號，超出單頁遊戲範圍，且會把輕鬆解謎變成競技壓力
- **體力 / 廣告解鎖系統**：留存道具對一頁式無意義，只會嚇跑路人玩家
- **特殊道具車（炸彈 / 傳送）**：增加規則理解成本，第一次玩就要讀說明，破壞「秒懂」的核心優勢

## 我們的變形版一句話企劃（史萊姆貓宇宙，核心樂趣要保留哪一條）

停車場被貓咪佔滿了！拖動橫衝直撞的毛球，替虹色史萊姆貓開出一條回家的路——**保留的核心樂趣：解開連鎖堵塞、看通道瞬間打開的那一刻豁然開朗感**。
