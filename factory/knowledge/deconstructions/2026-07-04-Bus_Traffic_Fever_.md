# 解構：Bus Traffic Fever!

## 核心迴圈（一圈幾秒？操作→回饋→獎勵怎麼轉？）
一圈約 3~8 秒：看一眼停車場（一堆互相卡住的彩色巴士）→ 找出「路沒被擋住」的那台 → 點一下 → 巴士滑出去停到候車位 → 排隊人群中同色乘客小跑步上車 → 車滿載開走、候車位空出來。每一次點擊都有三段回饋疊在一起：巴士移動的滑順動畫（操作成功）、乘客咚咚咚上車（配對成功）、車開走+格位清空（資源釋放）。玩家的下一步永遠是「現在該放哪台出來」，決策點密集但每個決策很輕，所以手停不下來。

## 上癮機制（用心理學拆）
- **歸因於己**：關卡是確定性的謎題，贏了是「我看出了正確順序」，輸了是「我點錯順序」——沒有隨機性甩鍋對象，玩家會立刻重開想證明自己。
- **損失趨避 + 倒數壓迫**：候車位只有 3~5 格，每放錯一台車就吃掉一格「永久資源」。看著格子一格格被佔滿，緊張感是漸進的，不是突然死亡——玩家有時間「眼睜睜看著自己犯的錯逼近」，這比即死更痛也更想再來一次。
- **Near-miss 設計**：輸的時候常常是「只差一台車就解開了」，最後一格被佔滿的瞬間下一台救命車明明就在眼前。差一點 = 立刻重試。
- **連鎖爽感**：一台車開走 → 格位空出 → 卡住的車能動了 → 又載走一批人。一次正確決策引發 2~3 步骨牌，多巴胺是批發的。
- **視覺化的進度條**：排隊人龍就是進度條，每上車一批人龍肉眼變短，「快清完了」的感覺持續拉著你。

## 難度與節奏（新手活過前 15 秒、老手 2 分鐘後不無聊）
- 前幾關只有 2~3 色、車位排列一眼看穿，第一關幾乎不可能輸——先讓玩家學會「點車→接人→開走」的爽，再學會怕。
- 難度旋鈕有三顆，可以分開轉：顏色數（2→5 色）、車陣複雜度（互卡層數變深、出現要繞路的車）、候車格數（5 格→3 格）。逐關輪流轉，玩家每關都感覺「跟上一關不一樣難」。
- 老手節奏靠「隊伍預讀」：排隊人龍前段可見、後段漸隱，高手會多讀 5~6 個人來規劃出車順序，形成技巧天花板。

## 可偷的設計（我們的單檔小遊戲做得到的）
1. **候車格 = 生命條**：3~5 個暫存格取代血量，錯誤是「佔格」而非扣血，壓力漸進、輸法自責，單檔 HTML5 完全做得到。
2. **確定性謎題 + 關卡種子**：不用隨機性也上癮，每關就是一個固定佈局，輸了「再一次」按鈕零成本重開。
3. **三段疊加回饋**：出車動畫→乘客上車彈跳→滿載開走粒子，全部 CSS/Canvas 動畫可做，爽感成本極低。
4. **人龍當進度條**：把剩餘量畫成看得見的隊伍，比數字進度條情緒強十倍。
5. **卡位判定超簡單**：格子座標 + 直線路徑檢查（前方有沒有車）就是全部核心邏輯，一個函式搞定，適合 30 秒上手。

## 不可行的部分（原作有但我們該捨棄的，為什麼）
- **數百關卡地圖 + 章節主題**：內容量產不現實，我們做 10~15 關手調關卡 + 一個無盡模式就夠。
- **金幣/道具內購體系**（洗牌、加格位道具）：沒有變現需求，道具反而稀釋「歸因於己」的純度，砍掉。
- **3D 車輛模型與轉彎尋路**：改成 2D 俯視格子 + 直線滑出，判定簡化十倍，樂趣沒少。
- **每日任務/簽到留存鉤**：需要帳號與後端，我們的留存靠關卡本身與玩家評分迴圈。

## 我們的變形版一句話企劃（史萊姆貓宇宙）
一個被毛線球車塞爆的貓咪停車場：點擊沒被擋住的史萊姆貓車滑出去，接走隊伍裡同色的小貓乘客——候車墊只有四塊，放錯順序墊子就滿——保留的核心樂趣是「一次看對出車順序、連鎖載客全清的自我歸因爽感」。
