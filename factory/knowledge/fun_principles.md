# 遊戲好玩學 — SlimeCat 工作室設計聖經

> 工廠每次生產前都要重讀這份。目標：每一款都比上一款更懂「為什麼好玩」。

## 一、上癮的心理機制（為什麼停不下來）

1. **核心迴圈 (Core Loop) 要短**：操作 → 回饋 → 獎勵 → 再一次，一圈 < 10 秒。
   玩家的爽感頻率 = 迴圈轉速。2048 一圈 2 秒（滑→合併→分數跳）。
2. **「再來一局」摩擦力 = 0**：死亡到重開 < 1 秒、一個點擊。
   Flappy Bird 的天才不是難，是「死了馬上能雪恥」。重開要等 = 上癮迴路斷電。
3. **Near-miss 差一點心理**：最強的重玩驅動不是贏，是「差一點就贏」。
   2048 差一步合出 512、俄羅斯方塊差一行清空。Game Over 畫面一定要顯示「差 X 分破紀錄」——把失敗包裝成邀請。
4. **歸因於己**：失敗要讓玩家覺得「是我可以更好」，不是「遊戲雷我」。
   盤面是玩家自己造成的（2048）→ 上癮；隨機秒殺（不可預見的障礙）→ 刪 app。
   隨機性可以有，但要「看得見、來得及反應」。
5. **心流難度曲線**：前 15 秒保證不會死（早期成功），之後隨分數/時間爬升。
   太簡單=無聊、太難=焦慮，甜蜜點是「剛好比上一局的我強一點點」。
6. **Juice 多汁回饋**：同樣的邏輯，有 juice 好玩三倍。每個互動都要有視聽回應：
   音效（WebAudio 合成）、粒子、震動/squash-stretch、分數跳動、combo 字樣、破紀錄慶祝。
7. **簡單規則 × 湧現深度**：規則一句話講完，深度靠組合湧現。
   規則解釋超過 3 句 → 重新設計。
8. **雙目標結構**：短期目標（這一步消掉/躲過）＋長期目標（最高分/合出西瓜）。
   目標要「具象化可見」——合成大西瓜讓你看得到終點長什麼樣。

## 二、經典解構筆記（可偷的設計）

### 2048（合併數字）
- 規則一句話：滑動，相同就合併。
- 上癮引擎：合併瞬間的「咔」快感（指數獎勵：翻倍比 +1 爽）；永遠差一步（near-miss 常駐）；
  盤面全是自己的決策結果（歸因於己）；分數 + 最大磚雙目標。
- 可偷：指數成長獎勵、一步一決策的緊湊感、「還有救」的錯覺維持到最後一刻。

### Flappy Bird（單鍵閃避）
- 一鍵操作、局超短（30 秒）、分數刻度極小（3 分都值得炫耀）。
- 可偷：單指操作、死亡即重開、恥辱驅動（「不可能這麼難」→ 再一局）。

### Block Blast / 俄羅斯方塊系（消除）
- 上癮引擎：清空的「掃乾淨爽感」＋ combo 連鎖；每一步都是小型解謎；
  盤面壓力節奏（緊→鬆→緊）製造呼吸感。
- 可偷：連鎖加成、消除的畫面+音效爆發、「差一格就大清版」。

### 合成大西瓜 Suika（合併＋物理）
- 上癮引擎：合併升級 + 物理不確定性（每次掉落都有小驚喜）；
  終極目標具象化（西瓜看得到）；失敗線逼近的張力（慢性壓力）。
- 可偷：物理帶來的意外感（但別讓物理「雷」玩家）、頂線警告的緊張節奏。

### Vampire Survivors / 割草類
- 上癮引擎：一局內從弱到神的成長曲線；升級三選一（小決策、大掌控感）；
  數值膨脹的視覺爽（滿屏傷害數字）。
- 可偷：局內成長、「選擇即獎勵」、生存時間當分數。

### 跑酷類（Subway Surfers 系）
- 上癮引擎：速度漸增的腎上腺素；障礙提前可見（歸因於己）；金幣路線=誘惑 vs 安全的抉擇。
- 可偷：速度=難度曲線、撿東西的誘惑設計、差一點撞上的驚險感。

## 三、單檔小遊戲硬規格（工廠出貨檢查清單）

- [ ] 標題畫面一句話講完規則，30 秒上手
- [ ] 一局 1~3 分鐘；重開 = 點一下，零等待
- [ ] 手機單指可玩（觸控優先），鍵盤加分
- [ ] 每個互動都有回饋：音效 + 視覺（WebAudio 合成，禁外部資源）
- [ ] 前 15 秒不會死；難度隨分數/時間爬升
- [ ] 即時分數 + localStorage 最高分；破紀錄要有慶祝演出
- [ ] Game Over 顯示「差 X 分破紀錄」或「新紀錄！」（near-miss 再利用）
- [ ] 失敗可歸因於玩家決策（不可有反應不及的隨機死）

## 四、AI 評審量表（每款出廠前自評）

| 維度 | 問題 | 1-10 |

| 上手 | 不看說明能玩嗎？規則一句話嗎？ | |

總分 < 30 → 列出前三個改進點，記進 learnings 給下一款。

## 五、迭代原則

- 玩家評分 > AI 自評 > 理論。玩家說不好玩，理論再對都要改。
- 每次生產前讀最近的 learnings（評分、點評、教訓），明確寫出「這一款比上一款好在哪」。
- 同類機制別連做兩款（多樣性也是留存）。
- 未來有真實數據（點擊率/留存率）後，用數據回頭修正這份文件本身。
